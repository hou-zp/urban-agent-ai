# MVP 任务分解清单

本文档将实施计划拆解为可分派、可跟踪、可验收的任务。任务状态建议使用：`todo`、`in_progress`、`blocked`、`review`、`done`。

## 1. 任务编号规则

| 前缀 | 领域 |
| --- | --- |
| PM | 项目管理与需求 |
| BE | 后端 |
| FE | 前端 |
| DB | 数据库与迁移 |
| AI | 智能体与模型 |
| QA | 测试与验收 |
| OPS | 部署与运维 |

优先级：

- P0：MVP 阻断项，必须完成。
- P1：MVP 核心项，建议完成。
- P2：增强项，可进入试点优化。

## 10. 完成定义

单个任务完成需满足：

- 代码已实现并自测通过。
- 必要单元测试或集成测试已补充。
- API 或页面行为与文档一致。
- 无硬编码密钥、无未处理异常、无敏感日志。

- 阶段验收用例全部通过。
