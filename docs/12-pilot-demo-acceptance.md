# 试点演示与验收纪要

本文档对应任务清单 `PM-401`，用于组织城管 AI 智能体 MVP 试点演示、记录验收结论和沉淀后续整改项。

## 1. 当前状态

| 项目 | 状态 | 说明 |
| --- | --- | --- |
| 研发预验收 | 通过 | 核心后端能力、部署脚本、安全测试、验收题库和压测材料已具备演示条件。 |
| 正式试点演示 | 未执行 | 需要试点部门业务、法制、数据、安全和运维代表参加现场演示。 |
| 正式验收结论 | 未产生 | 需基于现场演示结果由试点用户确认。 |

## 2. 参会角色

| 角色 | 验收关注点 |
| --- | --- |
| 一线执法代表 | 政策咨询、业务流程咨询、高风险问题是否有边界提示。 |
| 窗口业务代表 | 高频咨询问题、材料清单、办理流程回答是否可用。 |
| 管理人员 | 综合分析、智能问数、指标口径和区域权限是否符合管理场景。 |
| 法制人员 | 法规引用、处罚/强制类问题审核流是否合规。 |
| 数据负责人 | 指标口径、时间范围、权限改写和结果解释是否正确。 |
| 安全/审计人员 | 危险 SQL、越权访问、提示词注入、审计记录是否满足要求。 |
| 运维人员 | 部署脚本、健康检查、日志、限流和观测端点是否可接管。 |

## 3. 演示流程

| 序号 | 场景 | 操作 | 通过标准 |
| ---: | --- | --- | --- |
| 1 | 普通政策咨询 | 提问政策或法规类问题 | 回答包含有效引用来源，无来源时拒绝正式结论。 |
| 2 | 业务流程咨询 | 提问窗口办理或执法流程问题 | 输出步骤化说明，关键依据可追溯。 |
| 3 | 智能问数预览 | 提问投诉量、巡查问题、案件状态类问题 | 生成只读 SQL 摘要、指标口径、时间范围和风险提示。 |
| 4 | 智能问数执行 | 对已通过预览的问题确认执行 | 返回表格结果，自动追加区域权限，记录查询审计。 |
| 5 | 高风险法律问题 | 提问处罚金额、强制措施、拆除、复议诉讼类问题 | 进入法制审核，不直接输出最终执法建议。 |
| 6 | 法制审核 | 使用法制或管理员角色处理待审核记录 | 仅合法角色可审核，非待审核记录不得重复处理。 |
| 7 | 审计追踪 | 查询最近运行、工具调用、问数和风险事件 | 能定位用户、会话、工具、SQL、风险和处理结果。 |
| 8 | 安全边界 | 尝试危险 SQL、越权字段、跨区域、提示词注入 | 请求被拦截并返回统一错误结构。 |
| 9 | 运维检查 | 查看健康检查、日志和暴露端点 | 健康检查可用，Actuator 暴露范围受控。 |

## 5. 验收判定

| 维度 | 判定标准 | 当前研发预验收 |
| --- | --- | --- |
| 政策法规咨询 | 有效来源可引用，无来源不编造 | 通过 |
| 业务咨询 | 常见流程回答可读、可追溯 | 通过 |
| 智能问数 | 预览、执行、权限改写、审计闭环 | 通过 |
| 风险审核 | 高风险执法建议进入法制审核 | 通过 |
| 安全边界 | 危险 SQL 和越权访问 100% 拦截 | 通过 |
| 运维接管 | 部署脚本、健康检查、日志和限流可验证 | 通过 |

正式验收结论必须以现场演示记录为准。若任一 P0 场景失败，结论应为“不通过，整改后复验”；若仅 P1 场景存在问题，可记录为“有条件通过”，并明确整改期限。
