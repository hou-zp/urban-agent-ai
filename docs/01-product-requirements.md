# 城管部门 AI 智能体平台产品需求

## 1. 产品定位

本产品面向城管部门内部用户，提供政策咨询、法律法规咨询、业务流程咨询、数据查询、智能问数和综合分析能力。系统采用 ReAct 推理-行动范式，使智能体能够拆解复杂任务、选择工具、检索权威知识、查询受控数据，并输出可追溯、可审计的结果。

MVP 同时提供轻量前端 UI，覆盖试点阶段的核心交互：智能体对话、知识文档管理、智能问数和审计简表。

## 2. 目标用户

| 用户角色 | 核心诉求 | 权限重点 |
| --- | --- | --- |
| 一线执法人员 | 查询法规依据、执法流程、类似案例 | 按所属区域和岗位访问知识与案件摘要 |
| 窗口/热线人员 | 生成群众可理解的政策和业务答复 | 可访问公开政策、办事指南、热线知识库 |
| 指挥中心人员 | 查询工单、巡查态势、热点区域 | 可访问授权范围内实时或准实时业务数据 |
| 科室管理人员 | 分析治理成效、生成简报 | 可访问本科室和授权区域统计数据 |
| 法制审核人员 | 审核法律适用、裁量依据、风险答案 | 可访问法规库、裁量基准、审核记录 |
| 系统管理员 | 配置知识、工具、模型和权限 | 管理配置，不默认拥有敏感业务明细 |
| 审计/纪检人员 | 查看 AI 调用、数据访问和风险事件 | 只读审计权限 |

## 3. MVP 范围

MVP 必须包含：

- 智能体对话：支持多轮咨询、流式输出、引用来源、工具调用摘要和风险提示。
- ReAct 执行：支持工具选择、观察结果、继续推理、最终回答。
- 计划管理：综合分析任务可创建计划、更新步骤状态、取消和恢复。
- 政策法规 RAG：支持文档入库、分块、向量检索、来源引用和版本过滤。
- 智能问数：支持自然语言生成 SQL，执行前必须校验、改写权限并审计。
- 法律风险控制：高风险问题追问事实或转法制审核，不直接给执法结论。
- 权限与审计：按用户、角色、部门、区域控制访问并记录全链路日志。
- 轻量前端 UI：对话页、知识文档页、智能问数页、审计简表页。

MVP 不包含：

- 自动执法决定、自动处罚、自动发送正式文书。
- 写入、修改、删除业务系统数据。
- 面向公众的互联网咨询入口。
- 移动 App、小程序、语音/图片/视频识别。
- 完整后台管理系统和复杂可视化大屏。
- 模型微调训练平台。

## 4. 核心场景

### 4.1 政策咨询

示例问题：

- 夜市摊点临时设置需要满足哪些条件？
- 最近关于户外广告牌匾管理的政策有哪些变化？
- 占道经营治理专项行动的工作要求是什么？

产品要求：

- 答案必须引用有效政策来源。
- 如存在多版本文件，默认使用有效且最新版本。
- 如政策口径冲突，需提示冲突来源和推荐依据。

### 4.2 法律法规咨询

示例问题：

- 流动摊贩拒不整改，执法依据是什么？
- 违法建设拆除前需要履行哪些程序？
- 同一违法行为能否同时适用多个处罚条款？

产品要求：

- 输出法律依据、适用条件、程序要求、风险提示。
- 涉及行政处罚、强制、拆除、查封扣押时，不直接替代执法决定。
- 事实不足时必须追问，不直接给处罚金额或定性结论。

### 4.3 业务咨询

示例问题：

- 建筑垃圾运输车辆备案流程是什么？
- 商户申请临时占道经营要提交哪些材料？
- 投诉噪声扰民的工单应该如何流转？

产品要求：

- 返回办理条件、材料、流程、时限、责任科室和系统入口。
- 同一事项可按角色输出不同话术：窗口口径、一线处置口径、领导汇报口径。

### 4.4 智能问数

示例问题：

- 查询本周各街道占道经营投诉数量。
- 列出今天超期未处置的工单，按所属中队分组。
- 最近 30 天违建案件新增、结案和超期数量分别是多少？

产品要求：

- 识别指标、维度、时间范围、区域和过滤条件。
- 缺少关键条件时先澄清。
- 返回统计口径、数据来源、查询时间、权限或脱敏说明。

### 4.5 综合分析

示例问题：

- 分析本月占道经营投诉高发区域，并给出治理建议。
- 对比上季度和本季度户外广告案件变化，找出异常街道。
- 生成一份本周市容环境治理简报。

产品要求：

- 先生成任务计划，再执行数据查询、知识检索、分析和报告生成。
- 结果包含关键发现、数据证据、法规或政策依据、原因分析、建议措施和限制说明。

## 5. 法律风险规则

| 风险等级 | 场景 | 系统行为 |
| --- | --- | --- |
| 低风险 | 法规条文释义、公开政策解释、一般办理流程 | 可直接回答，必须引用来源 |
| 中风险 | 执法依据、裁量标准、证据要求、程序要求 | 回答需包含适用条件和风险提示，并记录审计 |
| 高风险 | 处罚金额、强制措施、拆除、查封扣押、重大案件、复议诉讼 | 不直接给最终结论，必须追问事实或转法制审核 |
| 禁止自动回答 | 规避程序、伪造材料、绕过审批、泄露内部意见 | 拒绝回答，记录风险事件 |

强制转法制审核：

- 用户要求判断具体案件是否违法、是否处罚、罚多少钱、是否可以强拆。
- 问题涉及强制措施、重大罚款、听证、复议、诉讼。
- 依据存在版本冲突、适用层级冲突或地方规定与上位法可能不一致。
- 答案将用于正式文书、处罚决定书、案件审核意见或对外公开回复。

## 6. 首批数据源和指标

首批数据源：

- 热线/投诉工单系统
- 巡查事件系统
- 行政执法案件系统
- 市容秩序专项台账
- 商户/门前三包台账
- 渣土运输/建筑垃圾系统

首批指标：

- `complaint_count`：投诉工单量
- `overdue_work_order_count`：超期未处置工单数
- `on_time_disposal_rate`：按期处置率
- `patrol_event_count`：巡查事件数
- `case_new_count`：新增案件数
- `case_closed_count`：结案案件数
- `street_hotspot_rank`：热点街道排行
- `violation_type_distribution`：违法/问题类型分布

MVP 不开放个人身份信息明细自由查询、未经授权的全市级明细导出、跨部门原始库表直接查询和任何写操作。

## 7. 验收指标

| 类别 | 指标 |
| --- | --- |
| 准确性 | 政策法规咨询 Top3 检索命中率 >= 90% |
| 可追溯 | 政策法规类回答引用覆盖率 = 100% |
| 安全 | 危险 SQL 拦截率 = 100% |
| 权限 | 越权数据访问拦截率 = 100% |
| 性能 | 普通咨询 P95 <= 8 秒，简单问数 P95 <= 10 秒 |
| 可用性 | 试点期核心服务可用性 >= 99.5% |
| 审计 | 工具调用、SQL 查询、高风险回答审计覆盖率 = 100% |
| 用户体验 | 试点用户有效问题解决率 >= 80% |

验收测试集：

- 政策咨询问题不少于 100 条。
- 法律法规问题不少于 100 条。
- 业务流程问题不少于 80 条。
- 数据问数问题不少于 80 条。
- 综合分析任务不少于 20 条。
- 权限与安全问题不少于 50 条。
