# 实施计划

## 1. 总体节奏

MVP 建议按 4 个阶段实施，总周期约 12-14 周。每阶段都应形成可演示、可测试的增量。

| 阶段 | 周期 | 目标 |
| --- | --- | --- |
| Phase 1 | 2-3 周 | 工程底座、基础 UI、会话 API、模型适配 |
| Phase 2 | 3-4 周 | AgentScope 集成、ReAct、RAG、知识文档页面 |
| Phase 3 | 3-4 周 | 数据目录、完整 NL2SQL、SQL 校验、智能问数页面 |
| Phase 4 | 3 周 | 风险审核、审计、观测、权限、生产加固 |

## 2. Phase 1：工程底座与基础 UI

后端交付：

- Spring Boot 工程初始化。
- 统一响应、全局异常、参数校验。
- OpenAPI 文档和健康检查。
- 基础用户、角色、区域模型。
- 会话、消息、AgentRun、ToolCall 基础表。
- `ModelProvider` 抽象和一个模型适配。
- 非流式问答接口和 SSE 基础接口。

前端交付：

- Vue 3 + Vite 工程初始化。
- 基础布局、路由、API client。
- 智能体对话页骨架。
- 消息列表、输入框、发送、加载状态。
- SSE 流式展示基础能力。

验收：

- 可新建会话并发送问题。
- 可返回模型回答。
- 可记录 AgentRun。
- 前端可展示消息和错误提示。

## 3. Phase 2：Agent + RAG

后端交付：

- 集成 AgentScope Java。
- 封装 `UrbanManagementAgent`。
- 实现 ReAct 工具注册和工具调用审计。
- 实现文档上传、解析、分块、embedding、pgvector 入库。
- 实现 `knowledge_search` 和 `law_clause_search`。
- 回答强制包含引用来源。
- 文档状态流转：draft、indexing、active、expired、abolished、failed。

前端交付：

- 知识文档页。
- 文档上传和元数据表单。
- 文档列表和索引状态展示。
- 检索调试入口。
- 对话页展示引用来源和工具调用摘要。

验收：

- 上传文档后可索引。
- 政策咨询能返回有效来源。
- 废止或过期文档不会作为默认依据。
- 工具调用进入审计记录。

## 4. Phase 3：智能问数

后端交付：

- 数据源、数据表、字段、指标字典模型。
- `data_catalog_search`。
- `nl2sql_generate`。
- `sql_validate`。
- `readonly_sql_query`。
- SQL AST 校验、权限改写、脱敏、行数限制、超时控制。
- QueryRecord 审计。

前端交付：

- 智能问数页。
- 自然语言输入。
- SQL 预览。
- 口径说明和风险提示。
- 执行确认。
- 表格结果展示。

验收：

- “查询本周各街道占道经营投诉数量”可返回统计结果。
- 危险 SQL 被拒绝。
- 越权区域查询被拒绝或改写。
- 查询结果包含口径、来源、时间和脱敏说明。

## 5. Phase 4：风险、审核与生产加固

后端交付：

- 法律风险分级规则。
- 强制转法制审核。
- `legal_review` 状态机。
- `risk_assess` 和 `user_confirm`。
- 审计查询接口。
- OpenTelemetry、Actuator、结构化日志。
- 模型调用成本记录。
- 限流、降级、任务取消和恢复。

前端交付：

- 对话页展示风险等级、待审核状态、审核结果。
- 审计简表页。
- 工具失败、模型不可用、权限不足、SQL 拒绝的标准提示。

验收：

- “这个能罚多少钱？”触发事实追问或审核。
- 高风险答案未审核时不输出最终执法建议。
- 审计页可查询最近运行、工具调用、问数和风险事件。
- 模型不可用时进入降级提示。

## 6. 测试策略

单元测试：

- 风险规则。
- SQL AST 校验。
- 权限改写。
- 指标口径解析。
- 文档状态过滤。

集成测试：

- 会话创建和消息发送。
- SSE 事件。
- 工具调用审计。
- RAG 检索。
- 问数 preview 和 execute。
- 法制审核状态流转。

安全测试：

- 危险 SQL。
- 越权字段。
- 越权区域。
- 提示词注入。
- 系统提示词泄露。
- 敏感字段脱敏。

前端测试：

- 对话页发送和流式展示。
- 知识页上传和索引状态。
- 问数页预览和执行。
- 审计页筛选和查看详情。
- 各类错误状态展示。

## 7. 试点验收

验收问题集：

- 政策咨询不少于 100 条。
- 法律法规不少于 100 条。
- 业务流程不少于 80 条。
- 数据问数不少于 80 条。
- 综合分析不少于 20 条。
- 权限与安全不少于 50 条。

验收指标：

- 政策法规咨询 Top3 检索命中率 >= 90%。
- 政策法规类回答引用覆盖率 = 100%。
- 危险 SQL 拦截率 = 100%。
- 越权数据访问拦截率 = 100%。
- 普通咨询 P95 <= 8 秒。
- 简单问数 P95 <= 10 秒。
- 审计覆盖率 = 100%。

## 8. 风险与缓解

| 风险 | 缓解措施 |
| --- | --- |
| 知识库质量不足 | 强制元数据、文档状态、引用来源和人工审核 |
| NL2SQL 风险高 | AST 校验、权限改写、只读账号、preview/execute 分离 |
| 法律答案误用 | 高风险识别、事实追问、法制审核、答案边界提示 |
| 模型不可用 | 主备模型、纯检索降级、错误提示 |
| 性能不可控 | 工具超时、ReAct 轮次限制、异步任务、缓存 |
| 权限复杂 | MVP 内置权限模型，接口预留 OIDC，审计全覆盖 |
