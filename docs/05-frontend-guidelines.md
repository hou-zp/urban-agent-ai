# 前端规范

## 1. 前端目标

MVP 前端用于试点交互和能力验证，不做完整后台管理系统。页面应简洁、稳定、信息清晰，优先满足咨询、知识入库、智能问数和审计排查。

技术栈：Vue 3 + Vite + TypeScript + Pinia + Vue Router。

## 2. 页面范围

### 2.1 智能体对话页

能力：

- 新建会话、切换会话。
- 输入问题并发送。
- 支持 SSE 流式展示。
- 展示工具调用摘要、引用来源、风险等级、计划步骤。
- 支持取消当前任务、恢复未完成任务。
- 高风险答案展示审核状态。

主要区域：

- 左侧会话列表。
- 中间消息流。
- 右侧详情面板：引用来源、计划步骤、工具调用、风险事件。
- 底部输入框。

### 2.2 知识文档页

能力：

- 上传文档。
- 填写文档元数据。
- 查看文档列表和索引状态。
- 触发索引、重新索引、废止或失效。
- 发起检索调试。

必须展示字段：

- 标题、发文机关、文号、生效日期、失效日期、适用区域、主题标签、状态、更新时间。

### 2.3 智能问数页

能力：

- 输入自然语言问题。
- 调用 query preview。
- 展示候选 SQL、统计口径、授权范围、风险提示。
- 用户确认后执行查询。
- 展示表格结果、数据来源、查询时间、脱敏说明。

交互要求：

- preview 未通过时禁用执行按钮。
- SQL 文本默认折叠，仅展示摘要，避免普通用户误读。
- 超时、越权、危险 SQL 必须有明确提示。

### 2.4 审计简表页

能力：

- 查看最近 AgentRun、ToolCall、QueryRecord、RiskEvent。
- 支持按时间、用户、风险等级、工具名筛选。
- 查看请求摘要、结果摘要、错误原因。

限制：

- 不展示完整敏感字段。
- 不展示系统提示词、模型密钥、完整 SQL 明细给无权限用户。

## 4. API 调用规范

统一响应：

```ts
interface ApiResponse<T> {
  code: number
  data: T
  message: string
}
```

要求：

- API client 统一处理 baseURL、鉴权头、错误码、超时。
- 页面组件不直接拼接 URL。
- 所有请求类型定义放在 `types/` 或 API 文件内。
- 错误提示使用业务可理解文本，不展示后端堆栈。

## 5. SSE 事件规范

前端需支持事件：

- `message.delta`：追加回答文本。
- `tool.started`：展示工具执行中。
- `tool.finished`：更新工具结果摘要。
- `plan.updated`：更新计划步骤。
- `risk.pending_review`：展示待审核状态。
- `agent.completed`：任务完成。
- `agent.failed`：任务失败。

处理规则：

- SSE 断开时显示重连或重试提示。
- 用户取消后关闭当前 EventSource。
- 同一 runId 的事件必须按顺序归并到同一消息。

## 6. UI 风格

风格定位：政务业务系统，克制、清晰、可扫描。

要求：

- 不做营销页、复杂动效、装饰性大背景。
- 信息密度适中，优先表格、列表、标签、状态徽标。
- 高风险状态使用明确颜色和文字提示。
- 引用来源以可展开列表展示。
- 表单字段必须有清晰标签和校验提示。

## 7. 状态与错误展示

必须处理：

- 加载中。
- 空数据。
- 权限不足。
- 模型不可用。
- 工具调用失败。
- SQL 校验失败。
- 查询超时。
- 文档索引失败。
- 法制审核中。

错误提示示例：

- “当前用户无权查询全市明细数据，已限制为授权区域。”
- “SQL 校验未通过：仅允许只读查询。”
- “模型服务暂时不可用，已切换为纯检索模式。”

## 8. 前端验收

- 对话页可完成一次政策咨询并展示引用来源。
- 对话页可显示综合分析计划步骤。
- 知识页可上传文档并查看索引状态。
- 问数页可完成 preview 和 execute 两步。
- 审计页可查看最近运行和风险事件。
- 所有页面在 1366x768 分辨率下无明显遮挡和横向溢出。
